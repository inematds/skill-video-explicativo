🎚️ Instrução vs layout — os DOIS níveis

Toda customização do visual de um vídeo HyperFrames acontece em dois lugares distintos com escopos completamente diferentes. Confundir os dois é a principal causa de mudanças que "sumiram" ou que "afetaram tudo sem querer". Entender o modelo é o primeiro passo.

1
Instrução (global) ~/.claude/skills/video-explicativo/

O skill instalado no seu ambiente Claude Code. Contém dois arquivos que definem o padrão de todo vídeo novo que o Claude gerar a partir de agora:

references/house-style.md
Mood, paleta, fontes, proporção, regras de composição. Tudo em prosa. O Claude lê e interpreta ao gerar o build-index.mjs.
scripts/composition-template.mjs
O template de código com CSS completo, classes (.title, .h2, .scene…), overrides 9:16 e estrutura de cenas.
2
Layout (local) projeto/build-index.mjs

A cópia gerada no projeto específico. É o CSS/HTML concreto daquele render. Mexer aqui afeta só aquele vídeo — rápido, sem consequências globais.

Regra de ouro
Mexer na instrução muda tudo pra frente (todos os vídeos novos). Mexer no layout muda só aquele vídeo.
💡
Dica prática
Antes de editar qualquer arquivo, pergunte: "Quero que isso mude em todos os próximos vídeos ou só neste?" Se a resposta for "só neste", toque apenas o build-index.mjs do projeto. Se for "em todos", edite o skill instalado — e lembre de re-empacotar o .zip.
Conceitos-chave
Instrução
house-style + template = padrão global
Layout
build-index.mjs = CSS/HTML concreto local
Escopo
global vs local — sempre decide primeiro
Re-empacotar
obrigatório ao mudar o skill instalado

🔠 Tamanho dos elementos

O composition-template.mjs define tamanhos base para 16:9. Há um bloco de overrides 9:16 (prefixo body.v) que redimensiona tudo pro vertical. Conhecer as classes e os valores concretos é o que permite ajustar com precisão.

Tamanhos base — 16:9 (composition-template.mjs)
Classe Propriedade Valor 16:9 Override 9:16
.title font-size 172px 118px
.h2 font-size 92px 74px
.lead font-size 34px 30px
.folder width / height 520px / 360px 440px / 300px
.code width auto 940px
.file-card font-size / top / left 36px / 120px / 110px 30px / 110px / 96px
.subhead font-size 40px 34px
Bloco real do composition-template.mjs — classes base 16:9
/* composition-template.mjs — tipografia e cards base */
.title{font-family:Sora,sans-serif;font-weight:800;font-size:172px;line-height:.95;letter-spacing:-.03em;margin:24px 0}
.subhead{font-size:40px;color:var(--muted)}
.h2{font-family:Sora,sans-serif;font-weight:800;font-size:92px;line-height:1.05;letter-spacing:-.02em}
.lead{font-size:34px;color:var(--muted);margin-top:28px;line-height:1.4}
/* folder */
.folder{position:relative;width:520px;height:360px;margin:0 auto}
/* overrides 9:16 */
body.v .title{font-size:118px;margin:20px 0}
body.v .h2{font-size:74px}
body.v .lead{font-size:30px;margin-top:24px}
body.v .folder{width:440px;height:300px}
🔼
Como aumentar um elemento
Localize a classe no build-index.mjs do projeto e suba o valor. Para .title de 172px → 200px, por exemplo:
- .title{font-size:172px}
+ .title{font-size:200px}
/* e o override 9:16, se quiser manter a proporção: */
- body.v .title{font-size:118px}
+ body.v .title{font-size:138px}
Conceitos-chave
Base 16:9
valores padrão do template horizontal
body.v
prefixo que ativa overrides 9:16
Proporção
ajuste base e override na mesma escala
Localizar
grep a classe antes de editar

📐 Margens de segurança

O .scene é o container de cada cena. Seu padding define o espaço entre o conteúdo e a borda do frame — a "margem de segurança". Diminuir = full-bleed; aumentar = mais espaço respiratório.

16:9 (landscape)
.scene{
/* top right bottom left */
padding:120px 150px 150px;
}
Top 120px, laterais 150px, bottom 150px. Bom equilíbrio pra 1920×1080.
9:16 (vertical / Shorts)
body.v .scene{
/* top right bottom left */
padding:170px 70px 240px;
}
Top 170px (câmera face), laterais 70px, bottom 240px (polegar + UI). Nunca reduza o bottom abaixo de 180px em Shorts.
Legenda tem margem própria — separada do padding do .scene
.caption{bottom:64px} /* 16:9 */
body.v .caption{bottom:150px} /* 9:16 — acima da barra de UI */
Ao reduzir o padding do .scene pra ir full-bleed, a legenda pode vazar. Ajuste .caption{bottom} proporcionalmente.
⚠️
Atenção: overflow invisível
Reduzir demais o padding pode fazer elementos saírem do canvas sem que você perceba no preview. Sempre rode npx hyperframes inspect --samples 16 depois de mudar margens. O audit vai acusar overflow se detectar conteúdo fora dos limites.
✓ Boas práticas
  • ✓ Manter bottom ≥ 120px no 16:9
  • ✓ Manter bottom ≥ 180px no 9:16 (Shorts UI)
  • ✓ Ajustar .caption{bottom} ao mudar padding
  • ✓ Rodar inspect depois de toda mudança
✗ Evitar
  • ✗ Padding zero ou negativo
  • ✗ Ignorar a legenda ao ir full-bleed
  • ✗ Mudar só 16:9 e esquecer o override 9:16
  • ✗ Publicar sem rodar inspect
Conceitos-chave
.scene padding
120/150/150 (16:9) · 170/70/240 (9:16)
.caption bottom
64px 16:9 · 150px 9:16
overflow
invisível no preview, visível no inspect
full-bleed
reduz padding mas valida sempre

🏷️ Posição da marca

Às vezes você quer um elemento de marca persistente — um logo, um watermark, um badge — que apareça em todas as cenas sem entrar na lógica de cenas/tweens. O padrão é adicionar esse elemento no .bg-layer (camada persistente de fundo) com data-layout-ignore e posicioná-lo via CSS, com override body.v para o vertical.

ℹ️
A CTA já existe — não reinvente
A cena final (scene9) sempre exibe a CTA INEMA.CLUB como encerramento obrigatório do vídeo. Ela cobre a "marca final". O elemento persistente que estamos criando aqui serve para identificação discreta durante o vídeo — tipo um watermark de canto.
Snippet: elemento de marca persistente no .bg-layer
/* build-index.mjs — dentro do .bg-layer, após os elementos existentes */
/* 1. Adicione o elemento com data-layout-ignore */
<div id="brand-mark"
data-layout-ignore="true"
style="position:absolute;bottom:40px;right:50px;opacity:0.35;pointer-events:none;">
<img src="brand/logo.svg" width="120" alt=""/>
</div>
/* 2. CSS: posição base (16:9) + override 9:16 */
#brand-mark{
position:absolute;bottom:40px;right:50px;
opacity:.35;pointer-events:none;
}
body.v #brand-mark{
/* 9:16: marca no topo, centrada */
top:60px;bottom:auto;left:50%;right:auto;
transform:translateX(-50%);opacity:.45;
}
🔑
Por que data-layout-ignore é obrigatório
O auditor do HyperFrames mede a bounding box de cada elemento pra detectar overflow. Um elemento de marca fora dos limites padrão vai acusar falso positivo — a não ser que você marque com data-layout-ignore="true". Com esse atributo, o auditor ignora o elemento na contagem de overflow.
Conceitos-chave
.bg-layer
camada persistente — aparece em todas as cenas
data-layout-ignore
exclui do audit de overflow
body.v override
posição diferente no 9:16 (topo centrado)
CTA scene9
INEMA.CLUB já é o encerramento — não duplique

🔁 Pontual vs permanente

Antes de qualquer edição, decida o escopo. A pergunta é simples: esta mudança vale só pra este vídeo ou pra todos os próximos? A resposta determina qual arquivo você toca — e quanto trabalho isso vai dar.

P Pontual — 1 vídeo
Edita o build-index.mjs do projeto e re-renderiza.
✓ Rápido — 1 arquivo, 1 render
✓ Sem risco de afetar outros projetos
✓ Não exige re-empacotar o skill
✗ Próximo vídeo não herda a mudança
Quando usar
  • • Ajuste fino de cor pra uma campanha
  • • Teste de um novo layout de card
  • • Override de fonte pra um cliente
  • • Correção de proporção num frame específico
G Permanente — todos
Edita house-style.md + composition-template.mjs no skill instalado e re-empacota.
✓ Todos os vídeos novos herdam
✓ Mantém consistência de marca
✗ Exige re-empacotar o .zip
✗ Mais impacto — testar antes
Quando usar
  • • Mudar paleta de cores do canal
  • • Adotar nova tipografia padrão
  • • Adicionar elemento de marca em todos
  • • Ajustar margens globais de segurança
Grid de decisão: qual arquivo editar?
Situação build-index.mjs house-style.md composition-template.mjs
Mudar cor de um vídeo só ✓ — —
Mudar fonte padrão de todos — ✓ ✓
Aumentar .title neste projeto ✓ — —
Aumentar .title em todos os vídeos — — ✓
Adicionar marca persistente global — ✓ ✓
Mudar margens só deste vídeo ✓ — —
Conceitos-chave
Pontual
build-index.mjs → re-render → pronto
Permanente
skill instalado → re-empacotar → todos herdam
Re-empacotar
sempre após editar o skill global
Testar antes
valide num projeto de teste antes do global

💬 Pedindo a mudança

O Claude Code entende linguagem natural, mas prompts bem estruturados produzem resultados mais precisos e previsíveis. Aqui estão templates prontos pra colar, com as informações que o modelo precisa pra saber onde e quanto mexer — e como validar.

Prompt 1 — ajuste visual pontual
/* cole no Claude Code — escopo: só este vídeo */
No próximo vídeo, edite o build-index.mjs deste projeto:
- Aumente as imagens: .folder de 520px → 620px (e override 9:16: 440px → 520px)
- Reduza margens: .scene{padding:80px 100px 100px} (9:16: padding:140px 50px 200px)
- Adicione marca no topo do 9:16: elemento #brand-mark em .bg-layer com data-layout-ignore, centrado, body.v #brand-mark{top:60px;left:50%;transform:translateX(-50%)}
- Depois rode: npx hyperframes inspect --samples 16 e corrija qualquer overflow
Prompt 2 — mudança global permanente
/* cole no Claude Code — escopo: todos os vídeos futuros */
Edite o skill instalado em ~/.claude/skills/video-explicativo/:
1. Em references/house-style.md: adicione regra "marca sempre no topo do 9:16, centrada, opacity 0.4"
2. Em scripts/composition-template.mjs: aumente .title de 172px → 192px e override 9:16 de 118px → 135px
3. Re-empacote o skill (.zip) e confirme que o SKILL.md lista as mudanças
Valide criando um vídeo de teste e rodando npx hyperframes inspect
Prompt 3 — flag configurável --brand-top
/* ideia avançada: parâmetro no build-index.mjs */
Adicione no topo do build-index.mjs:
const BRAND_TOP = process.argv.includes('--brand-top');
E no CSS do .bg-layer:
const brandCSS = BRAND_TOP ?
`body.v #brand-mark{top:60px;bottom:auto;left:50%;transform:translateX(-50%)}` : '';
/* uso: node build-index.mjs --brand-top */
🔍
Sempre validar com inspect + conferir frames
Após qualquer mudança de layout — pontual ou global — o fluxo de validação é sempre o mesmo:
npx hyperframes render # re-renderiza com as mudanças
npx hyperframes inspect --samples 16 # audita overflow, lint, timing
# Abra alguns frames no navegador e confira visualmente
# Só publique quando inspect retornar 0 problemas
✓ Prompt eficaz
  • ✓ Especifica o arquivo a editar
  • ✓ Diz a classe CSS e o valor novo
  • ✓ Menciona o override 9:16 se necessário
  • ✓ Pede pra rodar inspect no final
✗ Prompt vago
  • ✗ "deixa o título maior" — quanto? onde?
  • ✗ "muda as margens" — qual arquivo?
  • ✗ Não menciona se é pontual ou global
  • ✗ Não pede validação com inspect
Conceitos-chave
Especificidade
arquivo + classe + valor = prompt preciso
Escopo explícito
diga "só este vídeo" ou "todos os futuros"
--brand-top
flags configuram comportamento por execução
inspect sempre
0 problemas antes de publicar
