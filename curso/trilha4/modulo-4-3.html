🎬 Prompt base — do básico ao avançado

O ponto de partida de qualquer vídeo HyperFrames. Especifica assunto, progressão didática, ambos os formatos e a CTA final do INEMA.CLUB — tudo em um único pedido.

prompt-base.txt 📋 copiar prompt
Faça um vídeo explicativo sobre {ASSUNTO}, do básico ao avançado, em 16:9 e 9:16, terminando com a CTA do INEMA.CLUB. Use o skill video-explicativo. Gere roteiro completo em PT-BR com 6 a 8 cenas, narração via pf_dora --speed 0.98, paleta dark premium (#0D1321 base), tipografia Inter. A última cena deve ser a CTA padrão do INEMA.CLUB com o logo e a URL inema.club. Renderize os dois formatos.
O que ele faz

Aciona o skill completo: roteiro estruturado em cenas, narração TTS local com pf_dora --speed 0.98, animações CSS dark premium e dupla exportação (1920×1080 + 1080×1920). A última cena segue o template CTA padrão do INEMA.CLUB, com fade-in de 0.45s.

💡
Quando usar

Use como ponto de partida para qualquer assunto novo. Substitua {ASSUNTO} pelo tema — ex.: "Git rebase interativo", "como funciona o DNS", "introdução ao n8n".

Variações
Só um formato

Troque "em 16:9 e 9:16" por apenas "em 16:9" para economizar tempo de render quando não precisar do vertical.

Mais cenas

Adicione "com 10 cenas e exemplos de código" para conteúdo mais aprofundado — bom para tutoriais técnicos.

Tom específico

Acrescente "tom direto e sem jargão" ou "tom técnico para devs sênior" para calibrar a narração.

📱 Prompt Shorts vertical — hook forte, 60-90s

Para Reels e YouTube Shorts: formato exclusivamente 9:16, duração compacta (~60-90 segundos) e uma primeira cena que prende em menos de 3 segundos — o "hook".

prompt-shorts.txt 📋 copiar prompt
Crie um vídeo no formato Shorts/Reels sobre {ASSUNTO}. Só 9:16 (1080×1920). Duração total: ~75 segundos. A cena 1 deve ser um hook forte — uma pergunta ou afirmação impactante que apareça em menos de 3 segundos na tela, sem narração nos primeiros 2s. Use narração pf_dora --speed 1.02 (um pouco mais rápido). No máximo 5 cenas. Termine com a CTA do INEMA.CLUB.
O que ele faz

Gera apenas o vertical (--vertical ativado), com o build-index.mjs reescrevendo o canvas para 1080×1920. A narração usa pf_dora --speed 1.02 — levemente mais rápida que o padrão — e o roteiro é otimizado para retenção nos primeiros 3 segundos.

✓ Boas práticas para Shorts
  • ✓ Hook visual nos primeiros 2-3 segundos
  • ✓ Máximo 5 cenas — cada uma com propósito claro
  • ✓ Narração levemente mais rápida (speed 1.02)
  • ✓ Texto grande — legível no celular sem fone
✗ Erros comuns em Shorts
  • ✗ Começar com intro de logo lenta demais
  • ✗ Mais de 90 segundos — o algoritmo penaliza
  • ✗ Texto pequeno ou em área coberta pela UI do app
  • ✗ Recortar vídeo 16:9 — sempre gere nativo 9:16
💡
Quando usar

Ideal para divulgação rápida no Instagram Reels, TikTok e YouTube Shorts. Funciona muito bem com dicas únicas ("1 coisa que você não sabia sobre X") ou mini-tutoriais de uma etapa só.

🎨 Prompt mudar mood/paleta

Troque a identidade visual do vídeo sem reescrever o roteiro. Útil para criar versões com a cor de uma marca cliente, um tema claro (light mode) ou uma paleta completamente diferente.

prompt-paleta.txt 📋 copiar prompt
Refaça o vídeo sobre {ASSUNTO} usando uma paleta {PALETA} em vez do dark âmbar padrão. Mantenha o mesmo roteiro e narração já gerados. Altere apenas: cor de fundo base, cor de acento, cor dos textos secundários e gradientes. {PALETA} pode ser: "dark roxo com ciano" (bg #110a1f, acento #a855f7, ciano #38bdf8), "dark verde neon" (bg #0a1a0f, acento #22c55e, ciano #34d399), ou uma especificação hex personalizada. Renderize em 16:9 e 9:16.
O que ele faz

O skill substitui as variáveis de cor no composition-template.mjs e no CSS das cenas. O roteiro (narration-template.sh) é reutilizado, economizando o tempo de TTS. Apenas o render visual é refrescado.

🏁 Prompt trocar a CTA final

Substitua a cena final padrão do INEMA.CLUB pela marca ou URL do seu cliente — ou pela sua própria marca — mantendo todo o resto do vídeo intacto.

prompt-cta.txt 📋 copiar prompt
No vídeo sobre {ASSUNTO} que você gerou, substitua a cena final do INEMA.CLUB pela CTA da marca {MARCA_CLIENTE}. A nova cena final deve exibir: logo ou nome "{NOME_DA_MARCA}", URL "{URL_DA_MARCA}", e o texto de chamada "{TEXTO_CTA}" (ex.: "Acesse agora e comece grátis"). Mantenha o fade-in de 0.45s e o mesmo estilo dark premium das demais cenas. Renderize os dois formatos com a nova cena.
O que ele faz

O skill substitui apenas a última cena do projeto HTML. O FADE=0.45 e o TAIL=0.9 são preservados para consistência de timing. As demais cenas e toda a narração anterior são reaproveitadas — apenas a cena final é regravada e re-renderizada.

⚠️
Atenção ao uso comercial

Ao trocar a CTA para um cliente, certifique-se de ter os direitos sobre o logo e a identidade visual usados. O HyperFrames não valida propriedade intelectual — essa responsabilidade é sua.

✓ Boas CTAs finais
  • ✓ URL curta e legível (sem parâmetros UTM visíveis)
  • ✓ Texto de ação claro: "Acesse", "Experimente", "Saiba mais"
  • ✓ Logo em versão branca/transparente (fundo dark)
  • ✓ Manter fade-in 0.45s para não parecer abrupto
✗ CTA que não funciona
  • ✗ URL longa demais — impossível ler em 3 segundos
  • ✗ Logo colorido sobre fundo dark sem contraste
  • ✗ Texto passivo: "Confira se quiser" — sem urgência
  • ✗ CTA genérica sem relação com o tema do vídeo

⏱️ Prompt ajustar duração e ritmo

Controle preciso sobre o tempo total do vídeo — encurte para ~60s para distribuição social ou expanda para 3-5min para conteúdo educacional mais denso.

prompt-duracao.txt 📋 copiar prompt
No vídeo sobre {ASSUNTO}, ajuste a duração para ~60 segundos: reduza o número de cenas para 4, encurte a narração de cada cena para no máximo 12 segundos de áudio, use pf_dora --speed 1.05 (narração mais rápida). Mantenha LEAD=0.5 e TAIL=0.9 nas cenas, mas reduza o FADE para 0.3s entre cenas. Regenere a narração e renderize os dois formatos.
O que ele faz

Ajusta três variáveis de ritmo em cadeia: quantidade de cenas, velocidade TTS (--speed 1.05) e duração de transição (FADE=0.3). O LEAD=0.5 e TAIL=0.9 são mantidos para evitar cortes abruptos na narração — só o miolo é comprimido.

Parâmetros de timing do HyperFrames
Variáveis de cena
  • LEAD=0.5 — silêncio antes da narração (segundos)
  • TAIL=0.9 — silêncio após a narração (segundos)
  • FADE=0.45 — duração do cross-fade entre cenas
Velocidade TTS
  • --speed 0.95 — lenta, didática
  • --speed 0.98 — padrão INEMA.CLUB
  • --speed 1.02 — levemente mais viva
  • --speed 1.05 — compacto, Shorts
💡
Quando usar

Use quando tiver um vídeo já gerado mas o cliente pediu uma versão mais curta, ou quando quiser reutilizar o conteúdo em um formato diferente (ex.: vídeo de curso de 3min → teaser de 60s para o Stories).

🔧 Prompt regenerar só uma cena

Cirurgia de precisão: altere apenas uma cena específica sem reprocessar o vídeo inteiro — ideal para corrigir erro de narração, ajustar animação ou mudar o texto de uma slide.

prompt-regenerar-cena.txt 📋 copiar prompt
No vídeo sobre {ASSUNTO}, refaça só a cena {N} com a seguinte mudança: {DESCRICAO_DA_MUDANCA}. Mantenha todas as outras cenas e a narração delas intactas. Regenere apenas o áudio da cena {N} com pf_dora e renderize o vídeo final com a cena substituída. Exemplos de mudança: "corrija o texto 'HTTP' para 'HTTPS'", "adicione um diagrama de setas mostrando o fluxo", "mude o título da cena para 'Passo 2: Configuração'".
O que ele faz

O skill identifica o bloco HTML da cena N, aplica a mudança, regenera somente o .mp3 daquela cena via pf_dora --speed 0.98, e re-concatena o MP4 final via FFmpeg — sem tocar nas demais faixas de áudio ou frames renderizados.

✓ Mudanças boas para uma cena
  • ✓ Correção de erro ortográfico ou técnico no texto
  • ✓ Adicionar ou remover um elemento visual (seta, diagrama)
  • ✓ Encurtar a narração de uma cena que ficou longa demais
  • ✓ Trocar uma animação por outra mais clara
✗ Quando usar o prompt completo
  • ✗ Mudança de tom ou estrutura afeta todas as cenas
  • ✗ Reordenar cenas — impacta o áudio concatenado
  • ✗ Mudar paleta (use o prompt de paleta, tópico 3)
  • ✗ Adicionar cena nova no meio — renumera as demais
💡
Dica de eficiência

Se você errar em mais de 3 cenas, é mais rápido regenerar o vídeo inteiro com o prompt base corrigido do que aplicar 3 patches individuais. Use o patch de cena para 1-2 correções cirúrgicas.

✍️ Como escrever bons pedidos para o skill

Um prompt bem escrito reduz idas e vindas — e produz vídeos melhores na primeira tentativa.

1️⃣
Especifique o público

Diga "para devs iniciantes" ou "para gestores não-técnicos". O roteiro muda radicalmente dependendo de quem vai assistir.

2️⃣
Dê o objetivo do vídeo

"O espectador deve saber como configurar X ao final" é muito melhor que "explique X". Objetivo claro = roteiro focado.

3️⃣
Indique o formato destino

Diga explicitamente "16:9 e 9:16" ou "só 9:16". O skill não assume o formato — se você não disser, ele pedirá.

4️⃣
Mencione exemplos concretos

Se tiver um vídeo de referência ("no estilo do vídeo X"), descreva o que você gosta nele. Isso guia o tom das animações e da narração.

💎
Regra de ouro dos prompts

Se você consegue descrever o vídeo em uma frase clara — "um tutorial de 60s sobre Git rebase para devs que já conhecem Git básico, sem jargão, terminando com a CTA do INEMA.CLUB" — o skill consegue executar sem perguntas extras. Quanto mais específico, menos iterações.
